Known Issues
Using multiple browser tabs causing mis-allocation of marks and feedback
Opening Turnitin in more than 1 browser tab is not supported by Turnitin and can cause feedback and marks to be awarded to different students than were intended.
Feedback not visible to students
(e.g. error 124, error 127, error 9999 and object id not found)

Under certain circumstances, some students may not be able to access their feedback or mark. Most of these issues are caused by a synchronisation fault in Turnitin where it fails to update the KLE.

If you have students that cannot see their feedback, please use the Roster Sync button found within the Turnitin Assignment Inbox screen. This can only be performed by those with instructor access. Please try this before advising students to visit IT Connect. Guidance on how to do this can be found here.

Student work is not visible or students are missing within Turnitin
This is rare, but is caused by a synchronisation bug. Use the Roster Sync button and the missing work and/or students should appear, assuming the students have submitted their work. Guidance on how to do this can be found here.

Student submissions appear blank when opened in the Turnitin Document Viewer
This is caused by the new Windows 10 Edge browser and stops documents being uploaded correctly. The use of the Google Chrome browser is recommended and should help avoid this issue when submitting from a University PC. Edge is the default web browser for Windows 10 so only students with new personal laptops should ever see this problem.
Student's work is corrupt, with parts missing or badly formatted

As Turnitin use Microsoft Word 2010 to convert .doc and .docx documents to PDF (for the purposes of presenting submissions for marking), if a student has used Word 2013 or 2016, features that are not present in Word 2010 can cause the document to appear badly formatted or with parts missing.

If you suspect this to be the case, downloading the original document for checking is advisable. Further guidance can be found here.

Students cannot submit due to timeout
It has been known for students to click the Submit button and nothing to happen until a timeout screen appears. Although a specific workaround is not available, trying different PCs, browsers and waiting a few minutes is advisable.
